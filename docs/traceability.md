# Overview

ReguCompass AI is designed with:
# End-to-End Audit Traceability

Every compliance recommendation generated by the system can be traced back to:
- source documents
- clause references
- page numbers
- supporting evidence

This ensures:
- audit readiness
- governance explainability
- enterprise trust
- regulatory transparency

Every recommendation must follow:

```text
Recommendation
      ↓

      ↓

Page Reference
      ↓

Supporting Evidence
```

# Traceability Components

# 1. Page Mapping

Each chunk stores:
- page number
- document source
- metadata

Example:

```json
{
  "page": 4,
  "document": "RBI_Circular_2026.pdf"
}
```

---

# 2. Clause Identification

The system preserves:
- section hierarchy
- headings
- compliance clauses
- table structures

using:
```text
Marker + PyMuPDF
```

---

# 3. Semantic Evidence Linking

Every generated recommendation is matched against:
- retrieved chunks
- source embeddings
- compliance clauses

---

# 4. Auditor Validation

The Auditor Agent checks:
- citation validity
- clause existence
- evidence alignment
- semantic consistency
